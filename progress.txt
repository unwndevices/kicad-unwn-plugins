## 2026-07-09 — #21 Severity model + waiver sidecar (return-path.waivers.toml)

Added the §7 waiver system — the checked-in system of record for accepted
findings — on top of the #19 per-class severity config. Blockers #19 (config +
severity block) and #20 (full finding set to waive) were both closed.

- waivers.py (new): finding identity is a SHA-256 content hash of
  (check, class, net, layer, reference_layer) + the location snapped to a 0.5 mm
  grid (as integer cell counts, so float noise never shifts the key); severity/
  span/message are stored for the human echo but NOT hashed. So a cosmetic re-run
  is stable and a material change — defect moves past a grid cell, net renamed,
  reference layer swapped — alters the hash and the waiver lapses → re-review
  (the conservative KIID-rejection point of §7.2). apply_waivers stamps every
  finding with its id and carries waived ones (waived=True + reason, never
  dropped, §8.1); a waiver matching no current finding — or an expired one — is
  stale, surfaced as an info finding, never auto-deleted. Sidecar discovery
  mirrors config discovery (walk upward from the board; --waivers overrides;
  --no-waivers ignores). Hand-rolled TOML emit (like the sexpr emitter) for
  append (--waive) and full rewrite (--prune-waivers); provenance auto-stamps
  git user.name + the ISO date. Round-trip proven by load↔dump.
- detector.py: Finding gains three default-inert fields (id/waived/waiver_reason)
  at the end of the record; the detector never sets them, so its output is
  unchanged and existing keyword construction is unaffected.
- report.py: each active finding now carries its [id] (so a reviewer can copy it
  into --waive); waived findings move to a muted "Waived (N)" section and drop
  out of the active error/warning tally, mirroring the exit-code rule.
- cli.py: --waivers PATH / --no-waivers / --waive HASH --reason "…" /
  --prune-waivers. The exit code counts UNWAIVED findings only, so waiving the
  last active error greens the build (§10). --waive echoes a matching finding's
  full descriptive entry (or writes a bare id+reason when nothing matches this
  run) and can create a not-yet-existing explicit sidecar.

Post-implementation /code-review (high) — 7 finder angles + verification — found

construction site is keyword-based). Two edge findings fixed: (1) --waive to a

## 2026-07-09 — #20 Plane-edge clearance + return-via-at-layer-change checks

Added the remaining two v1 return-path checks beyond split-crossing (spec §5.1
#2/#3), completing the §5.1 check set. The #19 config layer had already reserved
the `edge_clearance_mm` / `return_via_distance_mm` thresholds and the
`edge_clearance` / `missing_return_via` severities, so this iteration was purely
the geometry + the two detector passes.

- stackup.py: new `PhysicalLayer` + `Stackup.build` parse the physical stack from
  `(setup (stackup (layer NAME (thickness …)) …))`; `dielectric_height(a, b)` sums
  the dielectric sub-layers between two copper foils — the per-trace H the §5.2
  edge-clearance formula needs. No `(setup (stackup …))` → empty build → H unknown
  (the 3H term drops out). `_parse_build` reads it; `parse_stackup` threads it.
- parser.py: new `Via` dataclass + `parse_vias` (net, x/y, copper-layer span) →
  `Board.vias`. The §3 numeric-net guard already rejects the pre-KiCad-10 form, so
  vias are name-based here too.
- detector.py: two passes wired into `check_return_path`.
  * Plane-edge clearance (`_edge_clearance`): for a trace a single plane *fully
    covers*, flag it when its distance to that plane's boundary falls below
    `max(3H, 90 mil, 1×trace width)` (or the scalar `edge_clearance_mm` override =
    flat floor). Only fully-referenced traces are in scope — a trace leaving the
    pour is the classifier's edge-overhang, so the checks never double-report. A
    trace well-referenced by *any* covering plane is clean.
  * Return-via (`_check_return_vias`): reference-net vias (GND/power) are the
    stitch candidates; each signal via that changes layers (≥2 distinct `.Cu`
    layers) with no reference via within `return_via_distance_mm` (default 2.0) is
    flagged. `signal_nets` now unions trace + via nets so a via-only net is still a
    victim. Both new classes resolve per-net/netclass thresholds and severities
    under §6 (edge-clearance = warning, missing-return-via = error) and honour the
    `ignore` off-switch; new `_message` cases + `_return_via_message`.
- edge_via_board.kicad_pcb (new fixture: 4-layer stackup with dielectric
  thicknesses + a GND plane + edge-hugging / comfortably-inside traces + a lonely
  signal via and a GND-stitched signal via) and test_returnpath_checks.py (13
  tests) cover all five ACs: the H-summed formula picks the dominant term (3H vs
  90 mil vs 1×W) and the scalar override replaces it; the hugging trace flags and
  the inside one doesn't; the lonely via flags and the stitched one doesn't; both
  checks respect `--set` threshold/severity overrides and `ignore`; and the two
  checks stay silent on the real split board (no false positives).
