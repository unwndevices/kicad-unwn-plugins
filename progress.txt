## 2026-07-09 — #18 Reference-plane identification + four-bucket classification

Turned the skeleton's single-GND-plane guess into real per-trace reference-plane
identification (spec §4), producing correctly-classified findings across every
signal net.

- stackup.py (new): parse `(layers …)` into physical copper order (F.Cu → In*.Cu
  → B.Cu by ordinal) + neighbour lookup — one for a microstrip (outer) layer, two
  for a stripline (inner) between planes (§4.3).
- parser.py: `reference_plane_refs` keeps net identity (one `PlaneRef` per
  (layer, net)) with the `min_pour_area_mm2` floor applied per plane, so a power
  pour qualifies and a sub-min sliver is dropped on its own merit (§4.2/§5.2);
  `parse_propagation` reads the declared Track-Propagation table
  (`(setup (track_propagation (layer … (top_reference …))))`, §4.1). `Board` gains
  `plane_refs`/`stackup`/`propagation` (defaulted → skeleton view untouched).
- detector.py: hybrid resolution — declared table wins, else stackup neighbours,
  else (no stackup) any other-layer plane; coverage always re-checked so a void in
  a *declared* plane is still caught. Four §4.4 buckets: solid (no finding),
  split-crossing (error, both-ends predicate), reference-change (info, covering
  plane's net/layer changes along a fully-referenced segment), edge-overhang/
  no-reference (warning). `check_split_crossing` kept as an alias.
- classify_board fixture (4-layer: slotted +3V3 power plane on In2.Cu, GND on
  F.Cu/B.Cu, sub-min GND sliver on In1.Cu) + inline boards drive all six ACs;
  15 new tests, skeleton's 19 unchanged.

Post-implementation /code-review (high) confirmed and fixed three items: a
GeometryCollection difference() result silently dropping a real void
(now flattened to its LineStrings); a reference-net trace able to self-reference
when the caller's `reference_nets` diverged from the parse set (now skips any net
that forms a plane); prep/edge caches re-keyed from object id() to (layer, net),
dropping a `type: ignore`. Refuted/kept as spec-intended: same-net layer-swap is a
reference-change per §4.4; the same-plane both-ends predicate is spec-mandated; the
span-area sliver floor is #17's resolved decision.

Verification: ruff + ruff format + mypy clean (50 files); 34 returnpath tests pass
(19 skeleton + 15 new); full non-GUI suite 743 green. The 1 red + 88 errored are
the pre-existing headless PySide6/libGL captouch-GUI failures, unchanged on the

## 2026-07-09 — #17 Walking skeleton: parser, split-crossing detector, text report, CLI

First end-to-end slice of the return-path checker: a real KiCad-10 board in,
split-crossing findings + an exit code out. New `packages/kicad-returnpath/`
package (hatchling, depends on kicad-core + shapely), the second consumer of the
carved-out sexpr parser.

- parser.py enforces the spec §3 contract: name-based nets everywhere
  (`(net "GND")`, no numbers, no zone `net_name` child); a single zone spans
  multiple layers (reference plane unioned *per filled_polygon layer*); islands
  taken verbatim (antipads baked in). A pre-KiCad-10 board (numeric nets /
  `net_name` child) is *rejected* (ParserContractError → CLI exit 2), never
  silently parsed to an empty plane + false "clean" pass — the #13 failure mode.
- detector.py: the §5.1 split-crossing check. `trace.difference(plane)` → drop
  spans below `min_crossing_span_mm` (0.1) and below the `sliver_ignore_area_mm2`
  copper floor (span length × trace width) → §4.4 both-ends-on-plane predicate
  labels survivors split-crossing (error) vs edge-overhang (warning). Plane
  prepared + buffered once (the spike's 25x speedup). Reference-plane
  *identification* proper (stackup adjacency, GND+power, propagation table) is
  deferred to #18; the skeleton checks each trace against every reference plane
  on a layer other than its own.
- report.py: default grouped, severity-iconed text report + error/warning tally.
- cli.py: `return-path check BOARD [--reference-nets …] [--fail-on …]`; exit
  codes 0 clean / 1 finding-at-or-above-fail-on / 2 usage-or-parse error.
- Root pyproject wired the new workspace member (uv sources + dev group, pytest
  pythonpath, ruff src, mypy files/mypy_path, coverage). kicad-core now exports
  `Node` (its public node type, now imported by the returnpath parser).
- tests/test_returnpath.py (19 tests) + two committed fixture boards (a 20260206
  split board, the legacy numeric-net board that must be rejected) cover all six
  ACs: reports real split-crossings (not a false empty-plane pass); rejects the
  legacy board; the both-ends predicate splits split-crossing from edge-overhang;
  grouped text report with a tally; exit codes 0/1/2.

Post-implementation /code-review (high) caught and fixed: (1) `parse_board`
hardcoded GND, silently ignoring `--reference-nets` → threaded through; (2) a
per-island sliver-area filter that contradicted §3 "verbatim" + the prototype →

## 2026-07-09 — #15 Per-tool PCM/release and repo rename

Generalized packaging + release from single-tool (captouch) to per-tool.

- packaging/build_pcm.py: introduced a `ToolSpec` record + `TOOLS` registry and a
  `--tool` flag. Everything previously hardcoded to captouch (identifier, display
  name, descriptions, archive basename, KiCad-version) now lives in the tool's
  registry entry, so packaging a new monorepo tool is a registry entry, not a code
  fork. captouch output is byte-identical (asserted via a pinned download_sha256
  regression test) and its reverse-DNS identifier is unchanged, so existing PCM
  installs are not orphaned (AC4). `repository.json` is now a repo-wide *shared*
  index ("unwndevices KiCad plugins"), which each tool release redeploys (AC3).
- .github/workflows/release.yml: triggers on per-tool tags `<tool>-vX.Y.Z`; a
  `setup` job resolves tool+version from the tag prefix and the captouch-specific
  binary/PCM jobs are gated on it, so one push builds only that tool (AC2). Added a
  `pypi` job that builds `packages/kicad-<tool>` and publishes via trusted
  publishing (OIDC, no stored token), with a tag-vs-pyproject version guard and a
  graceful skip for tools that have no buildable package yet. Pages tag policy
  comment updated to `*-v*`.
- Tests: byte-identical default build, arbitrary-ToolSpec parameterization,
  CLI `--tool`, identifier-unchanged. Full suite 858 green; ruff + mypy clean.

Repo rename (kicad-captouch → kicad-unwn-plugins) already landed on main in prior
commits (014d41b, 12e46e7), so AC1's reference-repointing is pre-existing; GitHub's
301 redirect of the old URL is operational.
